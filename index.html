How We Approached This Project

Below is a brief description of our methodology. You can also download a full project report [add link]

Labeling the Data

The data does not explicitly state whether or not someone is male or female so the team must solve this piece of data engineering first.

Data Clean-Up

Our team chose the Gender Guesser Python library that con- tains a list of more than 40,000 first names and gender, plus some 600 pairs of “equivalent” names. GenderGuesser labels data as “male/female”, “mostly male/female” and “unknown”. To ensure high quality of the data used in our prediction model, we only used unambiguous data labels.

Text Preprocessing using NLTK

The NLTK (The Natural Language Toolkit) is one of the most well-known language processing libraries in the industry. We eliminated stopwords, expanded contractions, stemmed the text, formatted and removed unwanted characters using regular expressions, and encoded the genders.

Applying our ML Model

We used convolutional neural networks to predict the gender with highest accuracy. The model consists of {xx} layers and uses relu and sigmoid activation functions.
